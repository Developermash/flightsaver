Политика конфиденциальности

Эта Политика описывает, какие данные использует приложение Flightsaver, зачем они нужны, где они хранятся и как пользователь может удалить свои данные.

Дата вступления в силу: 23 июня 2026 г. Контакт: developermash@mail.ru
Русский English

Flightsaver не продает персональные данные, не использует рекламный трекинг и не передает посадочные талоны рекламным сетям. Данные используются для работы функций приложения: хранения перелетов, предъявления посадочных талонов, синхронизации, погоды, карты, статусов рейсов и Live Activities.

1. Кто отвечает за обработку данных

Оператором приложения Flightsaver является разработчик приложения DeveloperMash. По вопросам конфиденциальности можно написать на developermash@mail.ru.

2. Какие данные хранит приложение

Flightsaver хранит данные, которые пользователь вводит вручную, импортирует из посадочного талона или получает через функции приложения.

История перелетов

Номер рейса, аэропорты вылета и прилета, дата, время вылета, авиакомпания, место, заметки, тип воздушного судна, регистрационный номер борта и другие сведения, которые пользователь добавляет или которые приложение получает для карточки рейса.

Посадочные талоны

Данные штрихкода и распознанные поля посадочного талона: пассажир, рейс, маршрут, дата, место, терминал, выход, зона, группа посадки, время посадки, номер брони, sequence number, данные программы лояльности, номера багажных бирок и технические поля, если они есть в талоне.

Настройки приложения

Выбранные параметры интерфейса, единицы измерения, статус включения iCloud-синхронизации, Live Activities и другие локальные настройки.

3. Где хранятся данные

  • Основная история перелетов хранится в локальной базе данных приложения на устройстве.
  • Посадочные талоны хранятся локально в хранилище приложения и используются для отображения посадочного талона внутри Flightsaver.
  • Если пользователь включает синхронизацию iCloud, история перелетов синхронизируется через Apple iCloud/CloudKit между устройствами пользователя.
  • Временные файлы, переданные через Share Extension, помещаются в общий контейнер приложения и удаляются после обработки или при очистке очереди импорта.

4. Геолокация

Flightsaver запрашивает доступ к геолокации "при использовании приложения". Геолокация нужна, чтобы показывать ближайший аэропорт, рассчитывать релевантность аэропортовых карточек и скрывать посадочный талон, когда пользователь уже находится в аэропорту прилета.

Текущая геопозиция обрабатывается приложением на устройстве. Flightsaver не отправляет точную текущую геопозицию пользователя на собственный авиационный сервер. При использовании системных служб Apple, например определения местоположения, обратного геокодирования, iCloud или WeatherKit, обработка данных также регулируется правилами Apple.

5. Камера, фото, PDF и файлы

Камера используется для сканирования штрихкодов посадочных талонов и распознавания текста. Фото, PDF и файлы используются только тогда, когда пользователь сам выбирает или передает их в приложение.

  • Кадры камеры, выбранные фото и PDF обрабатываются на устройстве.
  • Исходные фото и PDF не отправляются на авиационный сервер Flightsaver.
  • После импорта приложение сохраняет только данные, необходимые для истории перелета и отображения посадочного талона.
  • Если файл передан через системное меню "Поделиться", он временно копируется в общий контейнер приложения для обработки.

6. Sign in with Apple и iCloud/CloudKit

Sign in with Apple используется для локальной связи профиля приложения с Apple ID и включения синхронизации iCloud. Приложение может сохранить на устройстве идентификатор Apple user id, имя и email, если Apple передает эти данные приложению.

iCloud/CloudKit используется для синхронизации истории перелетов между устройствами пользователя. Данные в iCloud обрабатываются Apple и доступны пользователю в рамках его Apple ID. Если iCloud недоступен или пользователь не включает синхронизацию, приложение продолжает работать локально.

7. Live Activities и push-токены

Если пользователь включает Live Activities и функция доступна, приложение может создать Live Activity для рейса. Для удаленного обновления Live Activity на сервер Flightsaver передаются:

  • push-токен Live Activity;
  • идентификатор активности и внутренний идентификатор рейса;
  • номер рейса, дата, аэропорт вылета, аэропорт прилета и текст маршрута;
  • время вылета, терминал, выход, багажная лента и статус рейса, если эти данные доступны;
  • язык приложения и техническая среда push-уведомлений.

Эти данные используются только для обновления Live Activity через Apple Push Notification service. Серверная подписка удаляется при завершении активности, истечении срока действия или явном запросе на завершение. Максимальный срок серверной подписки Live Activity ограничен 36 часами.

8. WeatherKit

Flightsaver использует Apple WeatherKit для отображения погоды по аэропортам. Для получения прогноза приложение передает Apple координаты соответствующего аэропорта или выбранной точки прогноза. Имя пассажира, посадочный талон и номер брони не передаются в WeatherKit.

9. Авиационные данные через сервер Flightsaver

Для получения статусов рейсов, времени вылета, терминалов, выходов, сведений о борте и задержек аэропортов приложение обращается к серверу Flightsaver. Сервер скрывает API-ключи от приложения и адаптирует запросы к выбранному поставщику авиационных данных.

Сервер может использовать API.market/AeroDataBox, RapidAPI/AeroDataBox или FR24. Активный поставщик выбирается на серверной стороне. В запросах используются только данные, необходимые для работы функции: номер рейса, дата рейса, аэропорты, параметры маршрута и технические параметры запроса. Полные исходные файлы посадочных талонов, фото и PDF на авиационный сервер не отправляются.

Сервер может хранить технический кэш успешных ответов, чтобы ускорить работу приложения и снизить количество запросов к поставщикам данных. Кэш рейсов хранится в пределах рассчитанного срока жизни для конкретного рейса, кэш задержек аэропортов - до 2 часов, общий технический кэш - до 1 часа, если для конкретного запроса не задан меньший срок.

10. OpenStreetMap

Flightsaver использует тайлы OpenStreetMap для отображения карты маршрута. При открытии карты поставщик тайлов OpenStreetMap может получать технические данные сетевого запроса, включая IP-адрес, параметры устройства/браузерного клиента и координаты запрошенных тайлов. Эти запросы не содержат имени пассажира, номера брони или содержимого посадочного талона.

11. Передача данных третьим лицам

Flightsaver передает данные третьим лицам только в объеме, необходимом для работы выбранной функции:

  • Apple - для iCloud/CloudKit, Sign in with Apple, WeatherKit, геолокации, системных разрешений, Live Activities и push-уведомлений;
  • поставщикам авиационных данных через сервер Flightsaver - для получения данных о рейсах и аэропортах;
  • OpenStreetMap - для загрузки картографических тайлов.

Flightsaver не продает персональные данные и не использует их для рекламного профилирования.

12. Сроки хранения

  • История перелетов и посадочные талоны хранятся до удаления пользователем в приложении, удаления приложения или очистки соответствующих данных iCloud пользователем.
  • Локальные настройки хранятся до изменения или удаления приложения.
  • Данные Sign in with Apple хранятся локально до локального выхода из Apple-аккаунта в приложении или удаления приложения.
  • Live Activity-подписки на сервере хранятся только на время работы активности и не дольше 36 часов.
  • Серверный кэш авиационных данных хранится ограниченное время в соответствии с типом запроса.

13. Удаление данных

Пользователь может удалить отдельные перелеты и посадочные талоны внутри приложения. При удалении приложения локальные данные удаляются с устройства средствами iOS. Если была включена iCloud-синхронизация, данные, синхронизированные через iCloud/CloudKit, могут сохраняться в iCloud до удаления пользователем соответствующих данных через приложение, iCloud или системные инструменты Apple.

Для запроса по данным, связанным с серверной частью Flightsaver, можно написать на developermash@mail.ru.

14. Безопасность

Flightsaver использует системные механизмы iOS, iCloud/CloudKit и HTTPS-соединения для обмена данными с сервером и сторонними сервисами. Серверные API-ключи поставщиков авиационных данных не хранятся в приложении.

15. Дети

Flightsaver не предназначен специально для детей и не собирает сознательно данные детей. Если вы считаете, что ребенок передал персональные данные без согласия родителя или законного представителя, свяжитесь с нами.

16. Изменения политики

Мы можем обновлять эту Политику при изменении функций приложения, поставщиков данных или юридических требований. Актуальная версия всегда размещается на этой странице.
